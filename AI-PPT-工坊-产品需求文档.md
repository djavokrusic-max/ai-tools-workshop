## 六、技术方案

### 架构

```
用户浏览器 (Next.js 前端)
       ↓ HTTP
Node.js API (网站后端)
       ↓ 调用
Python 脚本 (pptx 生成)
       ↓
文件系统 (存储生成的 .pptx)
```

### 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Next.js（现有项目扩展） |
| 后端 API | Node.js / Next.js API Routes |
| PPT 生成 | python-pptx（需安装 Python） |
| 数据库 | SQLite（现有扩展） |
| 文件存储 | 服务器本地目录 /public/ppt |

### 数据库设计

#### users 表（已有）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| username | TEXT | 用户名（唯一） |
| password | TEXT | 密码（加密） |

#### ppt_generations 表（新增）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| user_id | INTEGER | 用户ID（关联users表） |
| outline | TEXT | 用户输入的提纲 |
| page_count | INTEGER | 页数 |
| style | TEXT | 风格 |
| file_path | TEXT | 生成的文件路径 |
| created_at | TEXT | 创建时间 |
| expires_at | TEXT | 过期时间 |
